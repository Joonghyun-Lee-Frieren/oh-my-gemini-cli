Features

Everything you need for agent orchestration

OmG extends Gemini CLI from a single-session tool into a full multi-agent orchestration system ‚Äî without forking. It uses only native extension points.

‚äö

Multi-Agent Orchestration

Spawn up to 6 concurrent agents ‚Äî Architect, Planner, Executor, Reviewer, Debugger, and Researcher ‚Äî working in parallel on your tasks.

‚óà

Context Engineering

Prompt caching optimization based on prefix stability, tool invariance, and cache-safe compaction. Includes JSON status telemetry for cache/context monitoring.

‚ñ¶

ASCII Dashboard

Real-time terminal UI built with Ink (React for CLI). See all agents, task pipelines, logs, and cache stats at a glance.

‚ü°

Dual Model Strategy

Gemini Pro for planning and architecture. Gemini Flash for fast code generation. Automatic routing based on task type with MCP-compatible orchestration hooks.

‚¨°

External LLM Support

Connect Claude, GPT, and other models via OAuth or API key. Mix and match models for different agent roles.

‚äï

Telegram / Discord Control

Monitor and control your agent sessions remotely. Get notifications on completion, errors, and review requests.

Quick Start

Up and running in 60 seconds

OmG is an add-on, not a fork. It uses only Gemini CLI's native extension points.

01 ‚Äî Install

Install globally from GitHub via npm

# Requires Node.js >= 20 and Gemini CLI
$ npm install -g github:Joonghyun-Lee-Frieren/oh-my-gemini-cli#main
02 ‚Äî Setup

Initialize in your project

# Installs prompts, skills, commands, MCP servers
$ omg setup
03 ‚Äî Verify

Check everything is ready

# Diagnose installation status
$ omg doctor
04 ‚Äî Run

Launch with multi-agent team

# Start a multi-agent session
$ omg team "Implement OAuth auth"

# Machine-readable status
$ omg status --cache --json
