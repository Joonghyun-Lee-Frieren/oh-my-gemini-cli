# $research — 병렬 리서치 스킬

웹 검색과 문서 분석을 병렬로 수행하여 빠르고 정확한 기술 조사 결과를 제공합니다.

## 트리거

`$research` 키워드로 활성화됩니다.

```
$research "Gemini API의 스트리밍 지원 현황을 조사해줘"
$research "Node.js ORM 비교: Prisma vs Drizzle vs TypeORM"
$research "MCP 프로토콜 최신 스펙과 변경사항"
```

## 사용 모델

**Gemini 3.1 Pro** (Google Search Grounding) — 실시간 웹 검색과 심층 분석에 적합합니다.

## 실행 프로세스

### Step 1: 질문 분해

복합적인 조사 요청을 독립적인 하위 질문으로 분해합니다:

```
입력: "Node.js ORM 비교: Prisma vs Drizzle vs TypeORM"

분해 결과:
  Q1: "Prisma ORM 최신 버전, 기능, 성능 벤치마크"
  Q2: "Drizzle ORM 최신 버전, 기능, 성능 벤치마크"
  Q3: "TypeORM 최신 버전, 기능, 성능 벤치마크"
  Q4: "Node.js ORM 비교 벤치마크 2025-2026"
```

### Step 2: 병렬 검색

분해된 질문을 **병렬로** 검색합니다:

```
          ┌── Q1: Google Search → 결과 수집
          │
$research ├── Q2: Google Search → 결과 수집
          │
          ├── Q3: Google Search → 결과 수집
          │
          └── Q4: Google Search → 결과 수집
```

### Step 3: 교차 검증

- 여러 소스의 정보를 대조합니다
- 상충하는 정보가 있으면 가장 신뢰도 높은 소스를 기준으로 합니다
- 날짜를 확인하여 최신 정보인지 검증합니다

## 검색 전략

### 소스 우선순위

1. **공식 문서** — docs, specification, RFC
2. **GitHub** — 코드, 이슈, 릴리스 노트, 벤치마크
3. **공식 블로그** — 공지, 로드맵, 마이그레이션 가이드
4. **Stack Overflow** — 실전 문제 해결 사례
5. **기술 블로그** — 경험담, 비교 분석

### 검색 팁

- 최신 정보가 중요한 경우 연도를 검색어에 포함합니다 (예: "2026")
- 벤치마크는 환경(OS, 버전)을 함께 확인합니다
- 라이선스 변경 이력이 있는 프로젝트는 현재 라이선스를 반드시 확인합니다

## 컨텍스트 엔지니어링 전략

- 검색 결과를 **요약하여 캐시**합니다 — 동일 주제 재검색 시 재사용
- 긴 문서는 관련 섹션만 발췌합니다
- 결과를 구조화된 JSON으로 저장하여 다른 에이전트가 참조할 수 있게 합니다

## 연계 스킬

- `$research` → `$plan`: 조사 결과를 기반으로 실행 계획을 수립합니다
- `$research` → `$execute`: 조사된 라이브러리를 즉시 도입합니다
